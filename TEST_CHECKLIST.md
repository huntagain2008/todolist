# TodoList 测试清单

## 功能测试 (Functional Testing)

- [ ] **任务创建成功** - 输入标题、优先级、截止日期后可成功创建
- [ ] **任务编辑成功** - 可修改标题、优先级、截止日期
- [ ] **任务删除成功** - 可删除指定任务
- [ ] **标记完成成功** - 勾选后任务状态变为已完成
- [ ] **状态筛选准确** - 全部/未完成/已完成筛选结果正确
- [ ] **优先级显示准确** - HIGH/MEDIUM/LOW 标签正确显示

---

## 边界测试 (Boundary Testing)

- [ ] **空标题无法保存** - 不输入标题点击添加时提示必填
- [ ] **超长标题警告** - 标题超过 200 字符时提示过长
- [ ] **无效日期拒绝** - 非法的日期格式被拒绝
- [ ] **删除后不重现** - 已删除的任务不会重新出现在列表中

---

## 用户体验测试 (User Experience Testing)

- [ ] **删除前确认** - 删除操作前弹出确认对话框
- [ ] **空列表引导** - 列表为空时显示友好提示
- [ ] **操作成功反馈** - 成功创建/编辑/删除后有即时反馈

## 测试数据示例

### 创建任务
```json
{
  "title": "完成项目文档",
  "priority": "HIGH",
  "dueDate": "2026-05-01"
}
```

### 更新任务
```json
{
  "title": "更新后的标题",
  "completed": true,
  "priority": "LOW",
  "dueDate": "2026-05-10"
}
```

### 筛选测试用例

| 筛选条件 | 预期结果 |
|----------|----------|
| `filter=all` | 显示所有任务 |
| `filter=uncompleted` | 仅显示 `completed=false` 的任务 |
| `filter=completed` | 仅显示 `completed=true` 的任务 |

### 边界值测试

| 字段 | 测试值 | 预期结果 |
|------|--------|----------|
| title | `""` (空) | 校验失败，提示必填 |
| title | `"a".repeat(201)` | 校验失败，提示过长 |
| priority | `"INVALID"` | 校验失败 |
| dueDate | `"not-a-date"` | 校验失败 |
| completed | `"true"` (字符串) | 校验失败 |
